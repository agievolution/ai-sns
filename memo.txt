二、技术实现
------------
1. 消息传递机制
   - 使用 window.postMessage 在 iframe 和父窗口之间通信
   - 消息格式：{ type: 'openSNSProfile', url: '...' }
   - 安全验证：检查消息来源（event.origin === 'http://localhost:8788'）

2. URL 规范化算法
   - 输入验证：检查 URL 是否为空或非字符串
   - 空格处理：trim() 去除首尾空格
   - 协议检测：使用正则表达式 /^https?:\/\//i
   - 智能判断：
     * 本地地址 → http://
     * 协议相对 URL → https:
     * 相对路径 → http://localhost:8788
     * 其他 → https://
   - 格式验证：使用 new URL() 验证最终 URL

3. DOM 操作
   - 动态创建页签按钮和内容面板
   - 使用 data-tab 属性标识页签
   - 使用 classList 管理激活状态
   - 使用 scrollIntoView 实现平滑滚动

4. 滚动优化
   - ResizeObserver 监听容器大小变化
   - 实时计算滚动位置和可滚动范围
   - 动态添加/移除 CSS 类控制渐变显示
   - 使用 CSS 过渡实现平滑动画

5. 事件处理
   - 关闭按钮点击事件（stopPropagation 防止冒泡）
   - 页签切换逻辑
   - 滚动事件监听
   - 窗口大小变化监听

五、URL 规范化示例
------------------
输入 → 输出：
- "example.com" → "https://example.com"
- "www.example.com" → "https://www.example.com"
- "localhost:8788" → "http://localhost:8788"
- "127.0.0.1:8080" → "http://127.0.0.1:8080"
- "192.168.1.100" → "http://192.168.1.100"
- "/path/to/page" → "http://localhost:8788/path/to/page"
- "//example.com" → "https://example.com"
- "  example.com  " → "https://example.com"
- "https://example.com" → "https://example.com"（保持不变）

十二、滚动条优化（v1.1.1）
--------------------------
更新时间：2026-01-29

问题：
- 滚动条太细（3px），用户难以拖拽
- 无轨道背景，不易发现滚动条
- 用户体验差

改进：
1. 滚动条高度：3px → 8px（增加 167%）
2. 添加轨道背景色：透明 → 浅灰色（#f5f5f5）
3. 添加滚动条边框：增加视觉层次
4. 悬停效果：滚动条变深色
5. 拖拽效果：滚动条变主题色（#1a73e8）
6. 渐变提示调整：bottom: 3px → 10px（避免遮挡滚动条）

技术实现：
- Webkit 浏览器：使用 ::-webkit-scrollbar 伪元素
- Firefox 浏览器：使用 scrollbar-width 和 scrollbar-color
- 支持悬停和激活状态的颜色变化

效果：
✅ 点击目标面积增加 167%
✅ 拖拽操作更加流畅
✅ 视觉反馈更加明显
✅ 用户体验显著提升
