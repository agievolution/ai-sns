🔗 URL 规范化测试

以下测试用例展示了不同格式的 URL 如何被自动规范化：

测试 1: 完整 URL（无需处理）
https://www.ai-sns.org
✓ 结果：https://www.ai-sns.org（保持不变）
测试 2: 缺少协议的域名
www.ai-sns.org
✓ 结果：https://www.ai-sns.org（自动添加 https://）
测试 3: localhost 地址
localhost:8788/test.html
✓ 结果：http://localhost:8788/test.html（自动添加 http://）
测试 4: 127.0.0.1 地址
127.0.0.1:8788
✓ 结果：http://127.0.0.1:8788（自动添加 http://）
测试 5: 局域网地址
192.168.1.100:8080
✓ 结果：http://192.168.1.100:8080（自动添加 http://）
测试 6: 相对路径
/profile/user123
✓ 结果：http://localhost:8788/profile/user123（转换为绝对路径）
测试 7: 协议相对 URL
//www.example.com/page
✓ 结果：https://www.example.com/page（添加 https:）
测试 8: 带空格的 URL
www.ai-sns.org
✓ 结果：https://www.ai-sns.org（自动去除空格）
